# 技术架构说明文档

## 1. 整体架构

LabFlow系统采用四层架构设计，从上到下依次为：GUI层、核心逻辑层、设备层和数据层。各层之间通过明确的接口进行通信，实现了高度的模块化和可扩展性。

### 1.1 架构层次
- **GUI层**：负责用户界面展示和交互
- **核心逻辑层**：负责工作流编排和执行
- **设备层**：负责设备抽象和控制
- **数据层**：负责数据持久化和管理

### 1.2 数据流
1. 用户在GUI层进行操作（拖拽节点、连接节点、配置参数）
2. GUI层将操作传递给核心逻辑层
3. 核心逻辑层处理工作流编排和执行
4. 执行过程中，核心逻辑层通过设备层控制实际设备
5. 设备层返回的数据通过核心逻辑层传递给GUI层显示
6. 工作流配置和执行结果通过数据层持久化

## 2. 模块功能和技术栈

### 2.1 GUI层
- **功能**：
  - 节点画布：支持节点拖拽、连线和参数编辑
  - 资源栏：显示可拖拽的节点库
  - 属性面板：显示和编辑节点属性
  - 日志面板：显示执行过程和结果
  - 数据可视化：实时显示实验数据
- **技术栈**：
  - PySide6：GUI框架
  - QGraphicsView/QGraphicsScene：画布和图形项管理
  - NetworkX：图形算法支持

### 2.2 核心逻辑层
- **功能**：
  - 工作流管理：添加、删除、修改节点和边
  - 执行引擎：基于拓扑排序的工作流执行
  - 数据传递：基于端口依赖的缓存传递
  - 状态管理：节点执行状态跟踪和反馈
- **技术栈**：
  - NetworkX：拓扑排序和图形分析
  - asyncio：异步执行支持
  - threading：线程池管理

### 2.3 设备层
- **功能**：
  - 设备抽象：统一设备接口
  - 设备管理：设备连接和状态管理
  - 协议支持：支持多种设备通信协议
  - 设备模拟：提供模拟设备用于测试
- **技术栈**：
  - 串口通信：serial
  - 网络通信：socket
  - 设备驱动：各设备专用驱动

### 2.4 数据层
- **功能**：
  - 工作流序列化：保存和加载工作流配置
  - 数据持久化：存储实验数据
  - 配置管理：系统配置和设备配置
- **技术栈**：
  - JSON：工作流序列化
  - 文件系统：数据存储
  - 配置文件：ini/yaml格式

## 3. 核心技术难点和解决方案

### 3.1 节点拖拽与连线
- **难点**：实现从左侧资源栏到画布的拖拽，以及节点间的连线
- **解决方案**：
  - 使用Qt的Drag & Drop机制实现节点拖拽
  - 自定义QTreeWidget子类处理拖拽数据准备
  - 实现QGraphicsItem的鼠标事件处理，支持连线操作

### 3.2 工作流执行引擎
- **难点**：实现基于依赖的工作流执行，处理复杂的依赖关系
- **解决方案**：
  - 使用NetworkX进行拓扑排序，确定执行顺序
  - 实现基于端口依赖的缓存传递机制
  - 使用asyncio和线程池实现非阻塞执行

### 3.3 实时数据可视化
- **难点**：实时处理和显示实验数据，确保UI响应性
- **解决方案**：
  - 使用PyQtGraph实现高效的实时绘图
  - 采用数据缓冲和批量更新策略
  - 使用线程分离数据处理和UI更新

### 3.4 多设备协同控制
- **难点**：统一不同设备的控制接口，处理设备通信的复杂性
- **解决方案**：
  - 设计统一的设备抽象接口
  - 实现设备驱动层，封装具体设备的通信细节
  - 提供设备模拟器，方便开发和测试
